# 要件定義書：Football Calendar

## 1. 開発目的

本アプリは、W杯開催にあたり、試合情報を自分で管理・閲覧できるWebアプリとして開発する。

また、就職活動におけるポートフォリオとして、単なるアプリ開発だけでなく、要件定義・設計・チーム開発・AI活用の過程も成果物として残すことを目的とする。

---

## 2. 開発背景

W杯をきっかけにサッカーを見始めるライトユーザーは、試合日程や出場選手、選手の所属クラブを把握しづらい。

そこで、W杯の試合情報をカレンダー形式で分かりやすく表示し、出場選手の所属クラブも確認できるようにすることで、W杯後にクラブリーグの試合へ興味を広げられるアプリを目指す。

---

## 3. 想定ユーザー

主な対象ユーザーは、W杯をきっかけにサッカーを見始める新規・ライトユーザーである。

---

## 4. MVPの範囲

最初に実装する最小機能は以下とする。

| 機能 | 内容 |
|---|---|
| 試合カレンダー表示 | W杯の試合日程をカレンダー形式で表示する |
| 試合詳細表示 | 対戦国、日時、会場、ステージを表示する |
| チーム別表示 | 特定の国・チームの試合だけを絞り込める |
| 選手情報表示 | 代表選手と所属クラブを表示する |
| データ保存 | 試合情報・選手情報を保存する |

---

## 5. MVPでは実装しない機能

以下は将来の拡張機能とし、初期開発では実装しない。

| 機能 | 理由 |
|---|---|
| iOS純正カレンダーへのコピー | 拡張機能として設計のみ考慮 |
| 通知機能 | MVP後に追加 |
| 試合結果の自動更新 | 初期は簡易構成を優先 |
| クラブリーグ日程表示 | W杯後の拡張として実装 |
| ユーザーアカウント機能 | MVPには不要 |

---

## 6. 非機能要件

| 項目 | 内容 |
|---|---|
| 保守性 | 機能ごとに責務を分離し、機能追加しやすい構成にする |
| 拡張性 | 将来的なカレンダー連携やリーグ追加を考慮する |
| 可読性 | READMEやコードを整理し、第三者が理解しやすくする |
| チーム開発 | GitHubでIssue・Pull Request管理を行う |
| AI活用 | 要件定義、設計、コードレビュー等にAIを活用する |

---

## 7. 管理方法

本アプリはGitHub上で管理する。

管理対象は以下とする。

- ソースコード
- README
- 要件定義書
- 設計書
- API仕様
- ER図
- Issue
- Pull Request
- AI活用ログ
- 開発振り返り

---

---

## 9. ポートフォリオで強調する点

本アプリでは以下を重視する。

- ライトユーザー向けUI設計
- MVPを意識した機能設計
- 将来拡張を考慮した設計
- GitHubを用いた二人チーム開発
- AIを活用した開発フロー
- W杯からクラブリーグへの導線設計
