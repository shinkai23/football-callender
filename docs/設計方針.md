## 選手・クラブ機能の設計方針

作成者: shinkai23

### 目的

W杯をきっかけにサッカーを見始めたライトユーザーが、代表選手の所属クラブを知り、W杯後にクラブリーグ観戦へ興味を広げられるようにする。

この機能では、試合日程を見るだけで終わらず、「この選手は普段どのクラブでプレーしているのか」を自然に確認できる導線を作る。

### 対象範囲

- チーム情報
- 選手情報
- 所属クラブ情報
- 試合詳細画面での選手・クラブ表示
- チーム別の選手確認

### MVPで扱うこと

- チーム一覧を取得できる
- 選手一覧を取得できる
- 選手ごとに所属代表チームを表示する
- 選手ごとに所属クラブを表示する
- 試合詳細から関連する選手情報を確認できる
- ライトユーザーが理解しやすい情報量に絞って表示する

### MVPで扱わないこと

- クラブリーグの日程表示
- クラブ詳細ページ
- 選手の詳細な成績
- 選手の移籍履歴
- 選手画像の管理
- 外部ニュース連携
- 外部サイトへの自動連携

### モデル設計方針

MVPでは、Team / Player / Club をそれぞれモデルとして扱う。

#### Team

W杯に出場する代表チームを表す。

主に以下の情報を持つ。

- チームID
- チーム名
- 国名

Player と Match から参照される想定とする。

#### Player

代表チームに所属する選手を表す。

主に以下の情報を持つ。

- 選手ID
- 選手名
- ポジション
- 所属代表チーム
- 所属クラブ

MVPでは、詳細な成績や移籍履歴までは持たない。

#### Club

選手が所属するクラブを表す。

MVPではクラブ名の表示を主な目的とするが、将来的にクラブリーグ日程やクラブ詳細ページへ拡張できるよう、文字列ではなくモデルとして管理する。

主に以下の情報を持つ。

- クラブID
- クラブ名
- 所属国またはリーグ名

### API設計方針

選手・クラブ機能では、Router / Service / Repository の責務を分ける。

- Router はリクエストとレスポンスの受け渡しを担当する
- Service は表示や取得に必要な処理の流れを担当する
- Repository はDBアクセスを担当する

MVPでは、以下のAPIを想定する。

- チーム一覧取得API
- 選手一覧取得API
- 選手詳細取得API

試合詳細画面で選手情報を表示する場合も、表示に必要な情報をAPIから取得できるようにする。

### UI設計方針

ライトユーザー向けに、情報を詰め込みすぎないことを優先する。

試合詳細画面や選手表示では、まず以下が分かる状態を目指す。

- どの代表チームの選手か
- 選手名
- ポジション
- 普段所属しているクラブ

クラブ情報は、詳細な説明ではなく「W杯後にこのクラブも見てみたい」と思える入口として表示する。

### 将来拡張の考え方

MVPではクラブリーグ対応を実装しないが、Club をモデルとして分離しておくことで、将来的に以下の拡張をしやすくする。

- クラブ詳細ページ
- クラブリーグの日程表示
- クラブ所属選手一覧
- クラブごとの試合情報

ただし、これらはMVP後の拡張機能として扱い、初期開発では実装しない。
