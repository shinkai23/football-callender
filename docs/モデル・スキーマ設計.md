# モデル・スキーマ設計

## 目的
SQLAlchemy モデルと Pydantic スキーマの責務を整理する。

## SQLAlchemy モデル
### Team
- id: int
- name: str
- country: str

### Club
- id: int
- name: str
- country: str
- league: str

### Player
- id: int
- name: str
- position: str
- team_id: int
- club_id: int

## Pydantic スキーマ
### TeamResponse
- id: int
- name: str
- country: str

### ClubResponse
- id: int
- name: str
- country: str
- league: str

### PlayerResponse
- id: int
- name: str
- position: str
- team: TeamResponse
- club: ClubResponse

## 設計方針
- SQLAlchemy モデルは DB 構造を表す
- Pydantic スキーマは API レスポンス形式を表す
- API では PlayerResponse に TeamResponse と ClubResponse を含める
- MVP では Create 系スキーマは作らない

## MVPで作らないスキーマ
- TeamCreate
- ClubCreate
- PlayerCreate
- PlayerStatsResponse
